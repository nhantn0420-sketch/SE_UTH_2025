# 3.1. PRODUCT OVERVIEW

---

## 3.1.1. Introduction

### System Name
**CollabSphere** - Hệ Thống Hỗ Trợ Học Tập Dựa Trên Dự Án (Project-Based Learning Management System)

### System Purpose

CollabSphere là một **nền tảng web tích hợp toàn diện** được thiết kế đặc biệt để hỗ trợ phương pháp học tập dựa trên dự án (Project-Based Learning - PBL) trong môi trường giáo dục đại học.

**Mục đích chính:**
1. **Tập trung hóa công cụ**: Thay thế việc sử dụng nhiều ứng dụng rời rạc bằng một nền tảng duy nhất
2. **Tăng hiệu quả cộng tác**: Cung cấp các công cụ real-time để sinh viên làm việc nhóm hiệu quả
3. **Quản lý dự án khoa học**: Hỗ trợ giảng viên theo dõi và quản lý nhiều dự án nhóm đồng thời
4. **Đánh giá minh bạch**: Theo dõi đóng góp cá nhân để đánh giá công bằng
5. **Hỗ trợ AI**: Tận dụng trí tuệ nhân tạo để hướng dẫn và tạo lộ trình dự án

### Target Users

CollabSphere phục vụ **5 nhóm người dùng chính**:

| User Role | Số lượng ước tính | Mục đích sử dụng chính |
|-----------|-------------------|------------------------|
| **Admin** | 1-2 | Quản trị hệ thống, xem báo cáo lỗi, quản lý toàn bộ users |
| **Staff** | 10-20 | Import và quản lý subjects, curricula, classes, student assignments |
| **Head of Department** | 3-5 | Phê duyệt projects, assign projects to classes, monitor teaching quality |
| **Lecturer** | 50-100 | Tạo projects, quản lý classes/groups, đánh giá students, facilitate collaboration |
| **Student** | 500-2000 | Làm việc nhóm, quản lý tasks, cộng tác real-time, nộp checkpoints, peer review |

### System Scope

CollabSphere bao gồm **8 module chính**:

![Figure 3.1.2: Module Structure Diagram](diagrams/3.1.2-module-structure.png)

*Figure 3.1.2: **CollabSphere Module Structure Diagram** - Hệ thống được tổ chức thành 3 tiers chính: **Core Tier** (nền tảng với 4 modules: User Management & Authentication, Academic Management, Project Management, Group & Workspace Management), **Collaboration Tier** (tương tác nhóm với 3 modules: Collaboration Tools bao gồm real-time chat/video/whiteboard, Evaluation & Feedback với peer review 4 chiều và grading, Resource Management với Cloudinary storage), và **Support Tier** (hỗ trợ với 1 module: AI Assistance & Notifications sử dụng AWS Bedrock Claude AI cho chatbot và auto-generate milestones). Các modules có dependencies rõ ràng: Core tier cung cấp foundation, Collaboration tier enables teamwork, Support tier triggers notifications across toàn bộ hệ thống.*

**8 Modules chi tiết:**
1. **User Management & Authentication**: Login, Registration, Profile, Role-based Access Control (JWT)
2. **Academic Management**: Subjects & Curricula, Class Management, Import from files, Student Enrollment  
3. **Project Management**: Create, Submit, Approve, Assign, Pick projects with AI-generated milestones
4. **Group & Workspace Management**: Teams (3-5 members), Milestones, Checkpoints, Tasks (Kanban board)
5. **Collaboration Tools**: Real-time Chat (Socket.IO), Video Calls (WebRTC), Whiteboard, Collaborative Editor
6. **Evaluation & Feedback**: Team evaluation, Peer review (4 dimensions: cooperation, contribution, communication, technical), Checkpoint grading (0-10), Final grade calculation
7. **Resource Management**: File upload/download (max 100MB), Cloudinary CDN storage, Document versioning
8. **AI Assistance & Notifications**: AI Chatbot (AWS Bedrock Claude), Auto-generate milestones, Real-time notifications (Socket.IO), Email notifications (SMTP)

## 3.1.2. Product Perspective

### System Context Diagram

![Figure 3.1.1: System Context Diagram](diagrams/3.1.1-system-context.png)

*Figure 3.1.1: **System Context Diagram** - CollabSphere Web Application (trung tâm) tương tác với 5 external actors (Admin quản lý users/subjects/curricula, Academic Staff import students và manage classes, Department Head approve projects và monitor progress, Lecturer create/evaluate projects, Student join groups và collaborate) và 5 external systems: **PostgreSQL Database** (store/retrieve toàn bộ application data qua SQL), **Cloudinary File Storage** (upload/download files qua REST API với max 100MB per file), **AWS Bedrock Claude AI** (generate milestones tự động và AI chatbot qua REST API), **SMTP Server** (send email notifications qua SMTP protocol), và **WebRTC** (peer-to-peer video/audio calls cho max 10 participants). Hệ thống core bao gồm React Frontend (SPA responsive), FastAPI Backend (RESTful API), và WebSocket Server (Socket.IO cho real-time features như chat và notifications).*

### System Interfaces

#### **1. User Interfaces**
- **Web Application**: Responsive Single Page Application (SPA) built with React
- **Supported Browsers**: Chrome 90+, Firefox 88+, Edge 90+, Safari 14+
- **Screen Resolution**: Minimum 1366x768, optimized for 1920x1080
- **Mobile**: Responsive design (tablet-friendly), no native mobile app

#### **2. Hardware Interfaces**
- **Client**: Any device with modern web browser (laptop, desktop, tablet)
- **Camera/Microphone**: Required for video conferencing features
- **Drawing tablet** (optional): For enhanced whiteboard experience

#### **3. Software Interfaces**

| External System | Interface Type | Purpose |
|----------------|----------------|---------|
| **PostgreSQL** | SQL Database | Store all application data (users, projects, groups, etc.) |
| **Cloudinary/S3** | REST API | Upload and retrieve files (documents, images, videos) |
| **AWS Bedrock** | REST API | AI chatbot for brainstorming and milestone generation |
| **SMTP Server** | SMTP Protocol | Send email notifications to users |
| **WebRTC** | P2P Protocol | Enable peer-to-peer video/audio calls |

#### **4. Communication Interfaces**
- **HTTP/HTTPS**: RESTful API communication between frontend and backend
- **WebSocket (Socket.IO)**: Real-time bidirectional communication for chat, notifications
- **WebRTC**: Peer-to-peer connection for video/audio streaming

## 3.1.5. Constraints

### Regulatory Constraints

1. **Data Protection**:
   - Must comply with Vietnam's Personal Data Protection regulations
   - User data (email, name) must be stored securely
   - Right to data deletion (GDPR-inspired)

2. **Educational Standards**:
   - Alignment with FPT University curriculum requirements
   - Grade and evaluation data retention for 5 years

3. **Accessibility**:
   - Basic compliance with WCAG 2.1 Level A (keyboard navigation, alt text)

### Technical Constraints

1. **Browser Compatibility**:
   - Must support Chrome 90+, Firefox 88+, Edge 90+, Safari 14+
   - No support for Internet Explorer

2. **Network Requirements**:
   - Minimum 1 Mbps for basic features
   - 2 Mbps for video conferencing

3. **Database**:
   - PostgreSQL 13+ (specific to this implementation)
   - No multi-database support

4. **File Upload**:
   - Maximum 100MB per file
   - Allowed types: PDF, DOCX, XLSX, PPTX, PNG, JPG, MP4

### Design Constraints

1. **Single Page Application**:
   - React-based SPA (no server-side rendering)
   - Requires JavaScript enabled

2. **Authentication**:
   - JWT-based (no session-based auth)
   - 24-hour token expiration

3. **Real-time Features**:
   - WebSocket required (Socket.IO)
   - Fallback to long-polling if WebSocket unavailable

### Budget and Resource Constraints

1. **Development Time**: 16 weeks (4 months)
2. **Team Size**: 4 developers
3. **Cloud Budget**: $200-500/month for infrastructure
4. **Third-party Services**:
   - AWS Bedrock: Limited free tier, then pay-per-request
   - File Storage: 100GB free, $0.10/GB/month additional

## 3.1.6. Assumptions and Dependencies

### Assumptions

1. **Internet Connectivity**:
   - Users have stable internet connection (minimum 1 Mbps)
   - No offline mode required

2. **Device Capabilities**:
   - Users have access to laptop/desktop with camera and microphone
   - Minimum screen resolution 1366x768

3. **User Training**:
   - Basic orientation session provided at semester start
   - Help documentation and tooltips sufficient for self-learning

4. **Data Accuracy**:
   - Staff imports correct and validated data for subjects/classes
   - Email addresses provided by users are valid and monitored

5. **Semester Structure**:
   - Fixed semester length (15-16 weeks)
   - Projects typically 8-12 weeks duration

### Dependencies

#### **External Dependencies**

1. **AWS Bedrock (Claude AI)**:
   - **Critical for**: AI chatbot, milestone generation
   - **Risk**: Service downtime or pricing changes
   - **Mitigation**: Implement graceful degradation (features work without AI)

2. **Cloudinary/S3**:
   - **Critical for**: File upload/download
   - **Risk**: Storage quota exceeded, service outage
   - **Mitigation**: Multiple storage provider support

3. **SMTP Server**:
   - **Critical for**: Email notifications
   - **Risk**: Email delivery failures
   - **Mitigation**: In-app notifications as primary, email as backup

4. **WebRTC Infrastructure**:
   - **Critical for**: Video conferencing
   - **Risk**: Firewall/NAT issues, limited peer connections
   - **Mitigation**: Use TURN/STUN servers, limit to 20 participants

#### **Internal Dependencies**

1. **User Data**:
   - Lecturers and students accounts must exist before class assignments
   - Subjects and curricula must be created before projects

2. **Academic Calendar**:
   - Semester start/end dates affect project timelines
   - Deadline reminders depend on milestone dates

3. **Team Formation**:
   - Students must be in teams before accessing workspace
   - Team size 3-5 members (configurable)

## 3.1.7. Operating Environment

### Client Environment

**Hardware:**
- **Processor**: Dual-core 2.0 GHz or higher
- **RAM**: 4 GB minimum, 8 GB recommended
- **Storage**: 500 MB for browser cache
- **Camera**: 720p webcam (for video calls)
- **Microphone**: Any USB or built-in microphone
- **Speakers/Headphones**: For audio communication

**Software:**
- **Operating System**: Windows 10+, macOS 10.15+, Linux (Ubuntu 20.04+)
- **Web Browser**: 
  - Chrome 90+ (recommended)
  - Firefox 88+
  - Microsoft Edge 90+
  - Safari 14+
- **JavaScript**: Enabled (required)
- **WebSocket Support**: Required for real-time features
- **WebRTC Support**: Required for video calls

### Server Environment

**Application Server:**
- **OS**: Linux (Ubuntu 22.04 LTS recommended)
- **Python**: 3.11+
- **FastAPI**: 0.104+
- **Uvicorn**: 0.24+ (ASGI server)

**Database Server:**
- **PostgreSQL**: 13+ with extensions (uuid-ossp)
- **Storage**: 50 GB minimum, 500 GB for large deployments

**File Storage:**
- **Cloudinary** or **AWS S3**
- 100 GB initial quota

**Caching (Optional):**
- **Redis**: 7.0+ for session management and caching

**Deployment:**
- **Docker**: 24.0+
- **Docker Compose**: 2.20+
- **Cloud Platform**: Azure, AWS, or Google Cloud

### Network Environment

- **Protocol**: HTTPS (TLS 1.2+)
- **Ports**: 
  - 443 (HTTPS)
  - 3000 (Frontend dev)
  - 8000 (Backend API)
  - WebSocket (same port as HTTPS)
- **Firewall**: Allow outbound HTTPS, WebSocket, WebRTC (UDP 3478-3479)
- **Load Balancer**: Nginx or cloud load balancer for production
