## 3.5.1. BUSINESS RULES

Business Rules (BR) định nghĩa các quy tắc nghiệp vụ mà hệ thống phải tuân theo.

### **Authentication & Account Management**

| ID | Business Rule | Category |
|----|---------------|----------|
| **BR-01** | Email address must be unique across the entire system | Validation |
| **BR-02** | Password must contain at least one uppercase letter, one lowercase letter, one number, and one special character | Security |
| **BR-03** | JWT token expires after 24 hours of inactivity; auto-refresh when user active | Security |
| **BR-04** | User can have only one primary role (cannot be both Lecturer and Student) | Authorization |
| **BR-05** | User role cannot be changed by the user themselves; only Admin can change roles | Authorization |
| **BR-06** | Super Admin role is reserved for initial system setup and cannot be assigned to regular users | Authorization |
| **BR-07** | Avatar image must be less than 5MB and in PNG/JPG format | Validation |
| **BR-08** | Changing password requires verification of current password | Security |
| **BR-09** | Admin cannot deactivate their own account | Business Logic |
| **BR-10** | Deactivated users' data remains in the system for 30 days before permanent deletion option | Data Retention |

### **Project Management**

| ID | Business Rule | Category |
|----|---------------|----------|
| **BR-11** | Only Admin can view all user accounts in the system | Authorization |
| **BR-12** | User passwords are never displayed in UI or logs | Security |
| **BR-13** | System logs (errors, activity) are retained for 90 days | Data Retention |
| **BR-14** | Only Admin can view system logs and error reports | Authorization |
| **BR-15** | Dashboard analytics data is cached for 5 minutes to improve performance | Performance |
| **BR-16** | Subject code must be unique and cannot be changed after creation | Business Logic |
| **BR-17** | Duplicate subject codes in import files are skipped with warning | Import Logic |
| **BR-18** | Subject code is immutable after creation (cannot be edited) | Data Integrity |
| **BR-19** | Subject cannot be deleted if it is linked to active classes or curricula | Data Integrity |
| **BR-20** | Head of Department can view all projects regardless of lecturer | Authorization |
| **BR-21** | Only projects with status "Pending" can be approved or rejected | Workflow |
| **BR-22** | Rejection reason is required when rejecting a project and must be visible to the lecturer | Workflow |
| **BR-23** | Approved projects are immutable unless explicitly updated by Head of Department | Workflow |
| **BR-24** | Chat messages cannot be edited after 5 minutes of being sent | Business Logic |
| **BR-25** | Only the message sender can delete their own messages | Authorization |
| **BR-26** | Chat message history is retained for 1 year | Data Retention |

### **Team & Collaboration**

| ID | Business Rule | Category |
|----|---------------|----------|
| **BR-27** | User can opt out of non-critical email notifications, but evaluation emails cannot be disabled | Notification |
| **BR-28** | Critical system emails (e.g., project approval, evaluation received) cannot be disabled | Notification |
| **BR-29** | Email delivery is retried up to 3 times before marking as failed | System Behavior |
| **BR-30** | In-app notifications expire and are auto-deleted after 30 days | Data Retention |
| **BR-31** | User can mark all notifications as read in one action | Usability |
| **BR-32** | Notification history is automatically purged after 30 days | Data Retention |
| **BR-33** | A team must have between 3 and 5 members (inclusive) | Validation |
| **BR-34** | Each team must have exactly one team leader | Business Logic |
| **BR-35** | A student can only be a member of one team per class | Business Logic |
| **BR-36** | A card in the workspace Kanban board can contain multiple tasks | Hierarchy |
| **BR-37** | A task can have multiple subtasks | Hierarchy |
| **BR-38** | Subtasks cannot have further children (maximum 3 levels: Card → Task → Subtask) | Hierarchy |
| **BR-39** | Team contribution percentages should sum to approximately 100% for the team | Business Logic |
| **BR-40** | Only team leader can mark milestones as complete | Authorization |

### **File & Resource Management**

| ID | Business Rule | Category |
|----|---------------|----------|
| **BR-41** | Maximum file upload size is 100MB per file | Technical Constraint |
| **BR-42** | Allowed file types: PDF, DOCX, XLSX, PPTX, PNG, JPG, MP4 (no executable files like .exe, .bat) | Security |
| **BR-43** | Total storage quota per organization: 100GB free, $0.10/GB/month for additional storage | Billing |
| **BR-44** | Files are stored in cloud storage (Cloudinary or AWS S3) with server-side encryption | Security |
| **BR-45** | Uploaded resources are immediately available to all authorized users in the class or team | Business Logic |

### **Evaluation & Grading**

| ID | Business Rule | Category |
|----|---------------|----------|
| **BR-46** | Peer evaluations are anonymous to other students (only visible to lecturer with full details) | Privacy |
| **BR-47** | Lecturer can see who submitted which peer evaluation | Authorization |
| **BR-48** | Student can only submit one peer evaluation per teammate | Business Logic |
| **BR-49** | Evaluations cannot be edited after final submission by lecturer | Data Integrity |
| **BR-50** | Students can view their evaluations only after lecturer has finalized and released them | Workflow |

### **AI & Automation**

| ID | Business Rule | Category |
|----|---------------|----------|
| **BR-51** | AI-generated content (milestones, suggestions) must be reviewed and approved by user before saving | Business Logic |
| **BR-52** | AI chatbot requests are limited to 50 per day per user to control costs | Resource Management |
| **BR-53** | AI chat responses are stored for future training and improvement (with user consent) | Data Usage |
| **BR-54** | If AI service is unavailable, system must gracefully degrade and allow manual input | Fault Tolerance |

### **Real-time Features**

| ID | Business Rule | Category |
|----|---------------|----------|
| **BR-55** | Video conferencing is limited to 20 participants per room due to WebRTC limitations | Technical Constraint |
| **BR-56** | WebSocket sync delay for collaborative features (whiteboard, editor) must be < 2 seconds | Performance |
| **BR-57** | If WebSocket connection fails, system falls back to HTTP long-polling | Fault Tolerance |
| **BR-58** | Real-time features require minimum 1 Mbps internet connection; video calls require 2 Mbps | Technical Constraint |

### **Data & Compliance**

| ID | Business Rule | Category |
|----|---------------|----------|
| **BR-59** | Personal data (email, name, phone) is only visible to authorized users based on role | Privacy |
| **BR-60** | Users have the right to request deletion of their personal data (GDPR-inspired) | Compliance |
| **BR-61** | Database backups are created daily and retained for 30 days | Data Protection |
| **BR-62** | Deleted projects and teams are soft-deleted (marked inactive) and can be restored within 30 days | Data Retention |
| **BR-63** | All timestamps are stored in UTC and converted to user's timezone on display | Data Format |

## 3.5.2. DATA DICTIONARY

### Core Entities

#### **User**
| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | UUID | Primary Key | Unique user identifier |
| `email` | String(255) | Unique, Not Null | User email address |
| `password_hash` | String(255) | Not Null | Bcrypt hashed password |
| `full_name` | String(100) | Not Null | User's full name |
| `avatar_url` | String(500) | Nullable | URL to user avatar image |
| `role` | Enum | Not Null | One of: Admin, Staff, Head, Lecturer, Student |
| `status` | Enum | Not Null, Default: Active | Active, Inactive, Pending |
| `created_at` | Timestamp | Not Null | Account creation timestamp |
| `last_login` | Timestamp | Nullable | Last successful login |

#### **Subject**
| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | UUID | Primary Key | Unique subject identifier |
| `subject_code` | String(10) | Unique, Not Null | Subject code (e.g., SE101) |
| `subject_name` | String(200) | Not Null | Subject full name |
| `description` | Text | Nullable | Subject description |
| `created_by` | UUID | Foreign Key → User | Staff who created |
| `created_at` | Timestamp | Not Null | Creation timestamp |

#### **Class**
| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | UUID | Primary Key | Unique class identifier |
| `class_code` | String(20) | Unique, Not Null | Class code (e.g., SE1701) |
| `class_name` | String(100) | Not Null | Class display name |
| `subject_id` | UUID | Foreign Key → Subject | Associated subject |
| `lecturer_id` | UUID | Foreign Key → User | Assigned lecturer |
| `semester` | String(20) | Not Null | Semester (e.g., Spring2025) |
| `start_date` | Date | Not Null | Class start date |
| `end_date` | Date | Not Null | Class end date |
| `created_at` | Timestamp | Not Null | Creation timestamp |

#### **Project**
| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | UUID | Primary Key | Unique project identifier |
| `title` | String(200) | Not Null | Project title |
| `description` | Text | Not Null | Project description |
| `subject_id` | UUID | Foreign Key → Subject | Related subject |
| `lecturer_id` | UUID | Foreign Key → User | Creator (Lecturer) |
| `status` | Enum | Not Null, Default: Pending | Pending, Submitted, Approved, Rejected |
| `difficulty` | Enum | Nullable | Easy, Medium, Hard |
| `estimated_duration_weeks` | Integer | Nullable | Estimated project duration |
| `rejection_reason` | Text | Nullable | Reason if rejected by Head |
| `created_at` | Timestamp | Not Null | Creation timestamp |
| `submitted_at` | Timestamp | Nullable | Submission for approval timestamp |
| `approved_at` | Timestamp | Nullable | Approval timestamp |

#### **ProjectMilestone**
| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | UUID | Primary Key | Unique milestone identifier |
| `project_id` | UUID | Foreign Key → Project | Parent project |
| `title` | String(200) | Not Null | Milestone title |
| `description` | Text | Nullable | Milestone description |
| `order` | Integer | Not Null | Display order (1, 2, 3...) |
| `duration_weeks` | Integer | Nullable | Expected duration |
| `created_at` | Timestamp | Not Null | Creation timestamp |

#### **Group (Team)**
| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | UUID | Primary Key | Unique group identifier |
| `class_id` | UUID | Foreign Key → Class | Parent class |
| `group_name` | String(100) | Not Null | Team name |
| `description` | Text | Nullable | Team description |
| `project_id` | UUID | Foreign Key → Project, Nullable | Assigned project |
| `created_by` | UUID | Foreign Key → User | Lecturer who created |
| `created_at` | Timestamp | Not Null | Creation timestamp |

#### **GroupMember**
| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | UUID | Primary Key | Unique membership identifier |
| `group_id` | UUID | Foreign Key → Group | Parent group |
| `user_id` | UUID | Foreign Key → User | Student member |
| `role` | Enum | Not Null, Default: Member | Leader, Member |
| `joined_at` | Timestamp | Not Null | Join timestamp |

#### **WorkspaceCard**
| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | UUID | Primary Key | Unique card identifier |
| `group_id` | UUID | Foreign Key → Group | Parent group/team |
| `title` | String(100) | Not Null | Card title (column name) |
| `position` | Integer | Not Null | Display order |
| `color` | String(7) | Nullable | Hex color code |
| `created_at` | Timestamp | Not Null | Creation timestamp |

#### **Task**
| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | UUID | Primary Key | Unique task identifier |
| `group_id` | UUID | Foreign Key → Group | Parent group |
| `card_id` | UUID | Foreign Key → WorkspaceCard, Nullable | Parent card (if level 2) |
| `parent_task_id` | UUID | Foreign Key → Task, Nullable | Parent task (if subtask) |
| `title` | String(200) | Not Null | Task title |
| `description` | Text | Nullable | Task description |
| `assigned_to` | UUID | Foreign Key → User, Nullable | Assigned member |
| `status` | Enum | Not Null, Default: Todo | Todo, InProgress, Done |
| `due_date` | Date | Nullable | Task deadline |
| `created_at` | Timestamp | Not Null | Creation timestamp |
| `completed_at` | Timestamp | Nullable | Completion timestamp |

#### **Evaluation**
| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | UUID | Primary Key | Unique evaluation identifier |
| `group_id` | UUID | Foreign Key → Group | Evaluated group |
| `student_id` | UUID | Foreign Key → User, Nullable | Evaluated student (if individual) |
| `evaluator_id` | UUID | Foreign Key → User | Lecturer or peer evaluator |
| `type` | Enum | Not Null | Team, Individual, Peer, Checkpoint |
| `grade` | Float | Nullable | Numeric grade (0-10) |
| `feedback` | Text | Nullable | Written feedback |
| `created_at` | Timestamp | Not Null | Evaluation timestamp |

#### **ChatMessage**
| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | UUID | Primary Key | Unique message identifier |
| `group_id` | UUID | Foreign Key → Group | Chat room (group) |
| `sender_id` | UUID | Foreign Key → User | Message sender |
| `content` | Text | Not Null | Message text |
| `file_url` | String(500) | Nullable | Attached file URL |
| `created_at` | Timestamp | Not Null | Sent timestamp |
| `edited_at` | Timestamp | Nullable | Last edit timestamp |
| `deleted_at` | Timestamp | Nullable | Soft delete timestamp |

#### **Notification**
| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | UUID | Primary Key | Unique notification identifier |
| `user_id` | UUID | Foreign Key → User | Recipient |
| `type` | Enum | Not Null | ProjectApproved, CheckpointSubmitted, EvaluationReceived, etc. |
| `title` | String(200) | Not Null | Notification title |
| `message` | Text | Not Null | Notification content |
| `link` | String(500) | Nullable | URL to navigate |
| `read` | Boolean | Not Null, Default: False | Read status |
| `created_at` | Timestamp | Not Null | Notification timestamp |

## 3.5.4. REQUIREMENT DEPENDENCIES

### Critical Path

```
1. Authentication (FE-01, FE-02) [MUST implement first]
   ↓
2. Basic User Management (FE-03, FE-05)
   ↓
3. Academic Structure (FE-08~16: Subjects, Classes, Users)
   ↓
4. Project Management (FE-17~26: Create, Approve, Assign)
   ↓
5. Team Management (FE-27~29: Create teams, Assign projects)
   ↓
6. Collaboration (FE-56~58: Tasks, FE-64: Chat)
   ↓
7. Evaluation (FE-40~43: Evaluate teams/members)
```

### Optional Features (Can be deferred)
- AI Features (FE-23, FE-32, FE-47, FE-63)
- Advanced Collaboration (FE-67 Whiteboard, FE-68 Collaborative Editor)
- Peer Review (FE-60, FE-61)
