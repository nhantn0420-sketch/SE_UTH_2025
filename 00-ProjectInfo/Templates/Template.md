Phần Mở Đầu (Front Matter)
•	Capstone Project Document (Tiêu đề chính của tài liệu).
•	Group Members (Danh sách thành viên nhóm): Bảng với cột Full Name, Role, Email, Mobile.
•	Supervisor (Người hướng dẫn): Tên và thông tin liên hệ.
•	Capstone Project Code (Mã dự án): Ví dụ: SP25SE107.
•	Table of Contents (Mục lục): Liệt kê tất cả phần với số trang.
•	Acknowledgement (Lời cảm ơn): Đoạn văn ngắn gọn, ký tên nhóm (ví dụ: Best regards, 4Bees).
•	Definition and Acronyms (Định nghĩa và viết tắt): Bảng với cột Acronym và Definition (ví dụ: BA - Business Analysis, ERD - Entity Relationship Diagram, SRS - Software Requirement Specification, v.v.).
I. Project Introduction (Giới thiệu dự án)
•	1. Overview (Tổng quan):
o	Project Information: Tên dự án, mã, tên nhóm, loại phần mềm (ví dụ: Web application).
o	Project Team: Bảng chi tiết thành viên và supervisor.
•	2. Product Background (Nền tảng sản phẩm): Mô tả vấn đề cần giải quyết.
•	3. Existing Systems (Hệ thống hiện có): Phân tích các hệ thống tương tự (ví dụ: Call4Project, Dev.to), bao gồm Description và Drawbacks (danh sách bullet points).
•	4. Business Opportunity (Cơ hội kinh doanh): Lợi ích của sản phẩm (ví dụ: Cải thiện collaboration giữa sinh viên và giảng viên).
•	5. Software Product Vision (Tầm nhìn sản phẩm): Mô tả ngắn gọn mục tiêu sản phẩm.
•	6. Project Scope & Limitations (Phạm vi và giới hạn):
o	Major Features: Phân loại theo role (Admin, Manager, Lecturer, Student) với mã FE-xx (ví dụ: FE-01: Login/Logout).
o	Limitations & Exclusions: Danh sách LI-xx (ví dụ: LI-1: Không verify sinh viên FPT đầy đủ).
II. Project Management Plan (Kế hoạch quản lý dự án)
•	1. Overview (Tổng quan):
o	Scope & Estimation: Bảng Work Breakdown Structure (WBS) với cột #, WBS Item, Complexity (Simple/Medium/Complex), Est. Effort (man-days). Bao gồm các giai đoạn như Initiating, Planning, Executing (với sub-items như Analysis, Design, Implementation, Testing, Monitoring).
•	2. Management Approach (Phương pháp quản lý): Mô tả cách tiếp cận (ví dụ: Agile hoặc Waterfall).
•	3. Project Deliverables (Sản phẩm dự án): Danh sách các deliverable (tài liệu, code, v.v.).
•	4. Responsibility Assignments (Phân công trách nhiệm): Ma trận RACI hoặc bảng phân vai trò.
•	5. Project Communications (Giao tiếp dự án): Kế hoạch họp hành, công cụ (email, Slack, v.v.).
•	6. Configuration Management (Quản lý cấu hình): Cách quản lý version code, tài liệu.
III. Software Requirement Specification (SRS - Đặc tả yêu cầu phần mềm)
•	1. Product Overview (Tổng quan sản phẩm): Tóm tắt chức năng chính.
•	2. User Requirements (Yêu cầu người dùng): Mô tả theo góc nhìn user (ví dụ: Use Cases).
•	3. Functional Requirements (Yêu cầu chức năng): Chi tiết các FE-xx, có thể dùng diagram hoặc mô tả step-by-step.
•	4. Non-Functional Requirements (Yêu cầu phi chức năng): Performance, Security, Usability, v.v.
•	5. Requirement Appendix (Phụ lục yêu cầu): Bảng Business Rules (BR), glossary, v.v.
IV. Software Design Description (SDD - Mô tả thiết kế phần mềm)
•	1. System Design (Thiết kế hệ thống): Architecture overview (ví dụ: High-level diagram).
•	2. Database Design (Thiết kế cơ sở dữ liệu): ERD, schema tables.
•	3. Detailed Design (Thiết kế chi tiết): Class diagrams, sequence diagrams, GUI mockups.
V. Software Testing Documentation (Tài liệu kiểm thử)
•	1. Scope of Testing (Phạm vi kiểm thử): Những gì sẽ test.
•	2. Test Strategy (Chiến lược test): Unit, Integration, System, UAT.
•	3. Test Plan (Kế hoạch test): Timeline, resources.
•	4. Test Cases (Trường hợp test): Danh sách chi tiết.
•	5. Test Reports (Báo cáo test): Kết quả, defects.
VI. Release Package & User Guides (Gói phát hành và hướng dẫn sử dụng)
•	1. Deliverable Package (Gói sản phẩm): Code, binaries, docs.
•	2. Installation Guides (Hướng dẫn cài đặt): Step-by-step.
•	3. User Manual (Hướng dẫn sử dụng): Hướng dẫn cho từng role.
